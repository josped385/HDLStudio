Extension API

Extend HDLStudio with Python plugins.

Overview

HDLStudio has a plugin system that lets you add custom functionality without modifying the core application. Extensions live in the extensions/ directory, each in its own subdirectory with a manifest.json and Python code.

Structure

textextensions/
  my_extension/
    manifest.json
    main.py
    ...

Manifest

FieldTypeDescription
idstringUnique identifier (e.g. "my_extension")
namestringHuman-readable name
versionstringSemantic version
descriptionstringShort description
enabledboolLoaded on startup if true

Example manifest.json:

json{
  "id": "hello_world",
  "name": "Hello World",
  "version": "1.0.0",
  "description": "A sample extension",
  "enabled": true
}

Extension Base Class

Your main.py must export a class that inherits from Extension (imported from core.extension_base) and implements at least on_load:

pythonfrom core.extension_base import Extension

class HelloExtension(Extension):
    def on_load(self, api):
        # Register a tool menu action
        action = api.register_action(
            "hello_world.say_hello",
            self.say_hello,
            label="Say Hello",
            tooltip="Prints Hello to the console"
        )
        api.add_tools_menu_action(action)
        api.print("Hello extension loaded!")

    def on_unload(self, api):
        api.print("Hello extension unloaded!")

    def say_hello(self):
        self.api.print("Hello from the extension!")

ExtensionAPI Reference

Actions

MethodDescription
register_action(id, callback, label, tooltip, icon_path)Register a named action with optional icon
add_tools_menu_action(action)Add action to the Tools menu
add_toolbar_action(action)Add action to the main toolbar
add_activity_bar_action(action, position)Add a button to the activity bar (position: "top" or "bottom")
add_context_menu_action(action, file_pattern)Add a right-click action in the file explorer (supports *.v, *.vhdl, etc.)
remove_action(action_id)Remove a previously registered action

UI Panels

MethodDescription
add_bottom_panel_tab(widget, title)Add a custom widget as a new tab in the bottom panel
remove_bottom_panel_tab(title)Remove a previously added tab

Console & Settings

MethodDescription
print(text)Write a line to the IDE console with the extension's prefix
settingsReturns a QSettings instance scoped to this extension

Extension Manager

The Settings > Extensions tab lists all discovered extensions and lets you enable/disable them individually. Disabling an extension calls its on_unload() hook and prevents it from loading on the next startup.

You can also open the extensions/ folder directly from the settings dialog.

Sample

A ready-to-use sample extension is included at extensions/sample_hello/. Enable it in the Settings dialog, then look for Say Hello in the Tools menu.
